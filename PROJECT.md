# WOLD-PM.com — Project Document
Last updated: July 1, 2026

## PROJECT OVERVIEW
WOLD-PM Post Meridian Radio — Ron's alt-universe pirate radio station.
Static site + Cloudflare R2-hosted audio, played via a single-page HTML
player (index.html) that reads a generated track list (tracks_local.js).
Repo: github.com/norsiwel/wold-pm

## ARCHITECTURE
- `index.html` — the player. One global `tracks[]` array, sequential/
  random playback, retry-then-skip error handling, stall detector.
- `tracks_local.js` — generated output, not hand-edited. Regenerated by
  `build_playlist_updated.py` and committed by `rotate_playlist.sh`.
- `music/` — regular rotation tracks (rglob'd for *.mp3)
- `bits/` — station IDs, standalone announcer bits, ads
- `shows/` — special multi-track music hours (DjangoHour, FolkInterlude,
  ArmstrongVol1/2, SciFiSpecial). Each show folder has its own
  `show.js` (or `show_vol*.js` for split volumes) listing {title, file}
  entries, generated by a per-show copy of `shows/_template/build_show.py`.
  `build_playlist_updated.py` reads these and splices them into the main
  playlist at evenly-spaced points.
- All audio served from Cloudflare R2 bucket `wold-pm`, base URL
  `https://pub-70b842d65a4f4ada82dec98f8d446fa2.r2.dev`

## SESSION — July 1, 2026

### Fixed: shows/ tracks never reached R2 (silent playback failures)
`rotate_playlist.sh` only uploaded `bits/*.mp3` to R2 — every show music
track (everything past the intro announcement) was never on the bucket.
Confirmed directly via `wrangler r2 object get ... --remote` returning
"The specified key does not exist." Matches the exact symptom in the old
`shows/SHOWS_STATUS.md`: shows play the intro, then fail on track 2+.

Fix: `rotate_playlist.sh` now also uploads everything under `shows/`,
tracked via a local manifest (`archive/.r2_shows_uploaded.txt`) so
already-uploaded tracks aren't re-pushed every run — with ~150 show
tracks and real `npx` overhead per upload, re-uploading unchanged files
every rotation would be slow and wasteful.

Also fixed a separate, related bug in `shows/_template/build_show.py`
(the generic template for NEW shows — not what built the existing ones,
those had their own per-show build scripts): it built R2 URLs via raw
f-string concatenation, no encoding. Any filename with spaces, curly
quotes, parens, or non-ASCII characters produces a broken URL. Now uses
the same `urllib.parse.quote()` approach `build_playlist_updated.py`
already used correctly.

First real run after the fix: 151 show tracks uploaded, playlist
regenerated and pushed (commit 391f779, then 60171f2 for the rotation
itself). Old broken `rotate_playlist.sh` archived at
`archive/rotate_playlist.sh.bak_2026-07-01_1858` (not deleted).

### Added: GoatCounter analytics
Cloudflare's built-in metrics were mostly bot/scraper traffic, not
useful for real listener counts. Added the same GoatCounter tracker
already running on readers-retreat.wold-pm.com — reusing the site code
`norsiwel.goatcounter.com` rather than provisioning a separate site, so
both domains report into one dashboard (Ron's explicit choice).
Added to `index.html` only (no `mobile.html` exists on this site, unlike
readers-retreat which has both).
